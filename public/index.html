API Developer Documentation

REST API Endpoints for MIT 8333 System Integration

Authentication API

POST /api/auth/register

Register a new student, staff, or officer.

Payload: { "name": "Maurice", "email": "maurice@miva.edu.ng", "password": "password123", "role_id": 3 }
POST /api/auth/login

Log in to receive JWT authorization token.

Payload: { "email": "admin@miva.edu.ng", "password": "admin123" }

Service Requests API

POST /api/requests

Create a new service request (Needs JWT Authorization header).

Payload: { "title": "Internet Down", "description": "No Wifi on 2nd floor", "category_id": 4 }
GET /api/requests

Fetch service requests. Automatically filtered based on the logged-in user's role (Students only see theirs, Officers see assigned, Admin sees all).

Query Params: ?search=wifi&status=Pending&page=1&limit=10
GET /api/requests/:id

Fetch detailed service request details along with full status logs and historical audit trail.

PUT /api/requests/:id/status

Update the status of a request (Admin or Assigned Officer only).

Payload: { "status": "In Progress", "remarks": "Replacing cables" }
POST /api/requests/:id/assign

Assign a request to a maintenance officer (Admin only).

Payload: { "officer_id": 2, "remarks": "Assigned to electric division" }

Administration API

GET /api/users

Fetch list of system users and their roles (Admin only).

GET /api/logs

Fetch full audit logs history (Admin only).

GET /api/reports/csv

Direct download for the system CSV database report (Admin only).
